## Homepage

Wenn deine App im Heimnetz laeuft, solltest du hier eher **keine** private NAS-URL eintragen.

Stattdessen kannst du das Feld leer lassen oder spaeter eine oeffentliche Projektseite verlinken.

## Social Preview

GitHub unterstuetzt fuer das Social Preview PNG, JPG oder GIF. Laut GitHub Docs wird mindestens `640 x 320` empfohlen, optimal ist `1280 x 640`.

## Empfohlene Reihenfolge auf GitHub

1. `README` und Doku pushen
2. Description setzen
3. Topics setzen
4. Social Preview Bild hochladen
5. spaeter optional License und Releases pflegen

## Hinweise zur Auffindbarkeit

GitHub-Repositories werden nicht wie klassische Webseiten ueber SEO gefunden. Wichtig fuer Auffindbarkeit auf GitHub sind vor allem:

- klarer Repository-Name
- gute Description
- passende Topics
- saubere README
- aktive Commits und Releases
